# yokoyama_site_update_v12

## 実装概要

横山未来子公式サイトの v11 を基準に、トップページのファーストビューを「自選短歌の一首」に再設計しました。短歌データと歌集データを `data/` 配下へ分離し、静的JSONを読み込む構成へ変更しています。

## 変更ファイル

- `index.html`
- `styles.css`
- `script.js`
- `mobile-preview.html`
- `README_v12.md`

## 追加ファイル

- `data/tanka.json`
- `data/books.json`
- `assets/profile-photo.jpg`（基準版から継承）
- `screenshots/`（ローカル確認用）

## 短歌データ外部化の仕様

- 短歌は `data/tanka.json` から読み込みます。
- 歌集・歌文集・著書は `data/books.json` から読み込みます。
- 短歌JSONの `html` には `ruby` / `rt` を含められます。
- 読み込み失敗時は console にエラーを出し、画面では fallback データを表示します。
- JSON読み込み確認はローカルサーバー経由を推奨します。
  - 例: `python3 -m http.server 8000`
  - `http://127.0.0.1:8000/`

## デザイン変更の要点

- ファーストビューを短歌中心へ再構成
- 短歌本文が viewport 中央に見えるように再調整
- 出典は本文の左側に縦書きで置き、下端が揃って見えるように絶対配置
- ボタン帯を短歌描画領域から分離
- ヘッダーを一定時間後に退避し、スクロール・スワイプ・マウス移動・タップ・キー操作で再表示
- 仮文言や確認用メモは画面上から整理

## レスポンシブ対応の要点

- 375 × 667
- 390 × 844
- 430 × 932
- 768 × 1024
- 1366 × 768
- 1440 × 900

を想定し、短歌本文と出典の位置が破綻しにくいようにフォントサイズと配置を実測ベースで調整します。
