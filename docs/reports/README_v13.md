# yokoyama_site_update_v13

## 実装概要
- v12 を基準に、短歌本文の表示演出を「短歌ブロック全体のフェード」ではなく「本文が1表示単位ずつ静かに現れる」方式へ変更しました。
- `ruby` / `rt` を壊さないよう、ルビ付き文字は1表示単位として扱っています。
- `prefers-reduced-motion: reduce` では逐字表示を無効化し、初回から全文表示にしています。
- 演出中に本文エリアをタップ / クリック / Enter / Space すると即時全文表示されます。

## 短歌データ外部化の仕様
- 正本データは `data/tanka.json` を使用します。
- `html` フィールドに `ruby` / `rt` を含められます。
- 読み込み失敗時は `script.js` 内の fallback データを使用します。
- JSON確認はローカルサーバー経由で行ってください。
  - 例: `python3 -m http.server 8000`
  - `http://127.0.0.1:8000/`

## 逐字表示の仕様
- 通常文字は1文字ずつ span 化して表示します。
- `<ruby>啓<rt>ひら</rt></ruby>` のようなルビ付き文字は 1 単位として扱います。
- 1単位あたり 35ms〜70ms 目安で表示します。
- 1首が極端に長くならないよう、単位数に応じて表示間隔を自動調整します。
- 出典は本文完了後に淡く表示します。
- 演出中に次の歌へ切り替えた場合、前のタイマーは破棄します。
